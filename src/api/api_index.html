Axiom Core API Reference

Content-addressable, version-controlled large object storage. All endpoints accept and return JSON unless noted otherwise.

v0.1 POC REST / JSON BLAKE3 Hashing Content Deduplication 18 Endpoints
Base URL
http://localhost:3000
Content Type
application/json
Hash Algorithm
BLAKE3 (64-char hex)
CORS
Open (all origins)

Conventions

Ref Resolution: Any path parameter that accepts a version ({id}, {ref}) will resolve in order: exact version ID → branch name → tag name.
Hash Format: All hashes are BLAKE3, represented as 64-character lowercase hexadecimal strings.
Error Responses: All errors return JSON with {"error": "message"} and an appropriate HTTP status code (400, 404, 500).

Versions

POST /api/v1/versions/ Create a new version

Create a new version (commit) pointing to an existing root tree hash. Optionally updates a branch ref to point to the new version.

Request Body
FieldTypeDescription
rootstring requiredBLAKE3 hex of the root tree hash
parentsstring[] requiredParent version IDs (empty array for initial commit)
messagestring requiredCommit message
branchstring optionalBranch to auto-update (e.g. "main")
metadataobject optionalKey-value metadata pairs
 
{
  "root": "abc123def456...",
  "parents": [],
  "message": "initial commit",
  "branch": "main",
  "metadata": { "author": "alice" }
}
{
  "id": "f8a9b1c2...",
  "root": "abc123def456...",
  "parents": [],
  "message": "initial commit",
  "timestamp": 1713500000,
  "metadata": { "author": "alice" }
}
cURL Example
curl -X POST http://localhost:3000/api/v1/versions/ \
  -H "Content-Type: application/json" \
  -d '{"root":"abc123...","parents":[],"message":"init","branch":"main"}'

PUT /api/v1/refs/{name} Update a branch

Update a branch to point to a new version. Only branches can be updated — tags are immutable.

Note: This only works for branches. Attempting to update a tag will return an error.
FieldTypeDescription
targetstring requiredNew target version ID
cURL Example
curl -X PUT http://localhost:3000/api/v1/refs/main \
  -H "Content-Type: application/json" \
  -d '{"target":"new_version_id..."}'
DELETE /api/v1/refs/{name} Delete a branch

Delete a branch. Only branches can be deleted — tags are immutable.

Note: This only works for branches. The underlying versions are not deleted.
Response · 200 OK
{
  "deleted": "feature-branch"
}
cURL Example
curl -X DELETE http://localhost:3000/api/v1/refs/feature-branch

Diff

POST /api/v1/diff/ Compute diff between versions

Compare two versions and produce a list of file-level changes (added, removed, modified) along with chunk-level deduplication statistics.

Request Body
FieldTypeDescription
old_versionstring requiredOld version ID, branch, or tag
new_versionstring requiredNew version ID, branch, or tag
 
{
  "old_version": "main~1",
  "new_version": "main"
}
{
  "entries": [
    {
      "path": "src/lib.rs",
      "kind": "modified",
      "old_hash": "aaa...",
      "new_hash": "bbb..."
    },
    {
      "path": "README.md",
      "kind": "added",
      "old_hash": null,
      "new_hash": "ccc..."
    }
  ],
  "added_files": 1,
  "removed_files": 0,
  "modified_files": 1,
  "added_chunks": 5,
  "removed_chunks": 3,
  "unchanged_chunks": 10
}
cURL Example
curl -X POST http://localhost:3000/api/v1/diff/ \
  -H "Content-Type: application/json" \
  -d '{"old_version":"main~1","new_version":"main"}'

Upload

POST /api/v1/upload/file Upload a single file

Upload a single file as raw bytes. The server chunks the file, deduplicates against existing content, builds a tree, creates a version, and optionally advances the branch.

Content-Type: Send the raw file bytes as the request body (application/octet-stream). File path and options are passed as query parameters.
Query Parameters
NameTypeDefaultDescription
pathstring required—Relative file path (e.g. src/main.rs)
branchstring optionalmainBranch to commit to
messagestring optionalupload {path}Commit message
parentsstring optionalauto (HEAD)Comma-separated parent version IDs
Response · 200 OK
{
  "version_id": "abc123...",
  "root": "def456...",
  "branch": "main",
  "stats": {
    "total_files": 1,
    "total_bytes": 4096,
    "total_chunks": 3,
    "new_chunks": 2,
    "dedup_chunks": 1,
    "dedup_bytes": 1024
  }
}
cURL Example
curl -X POST "http://localhost:3000/api/v1/upload/file?path=docs/readme.md&branch=main" \
  --data-binary @readme.md
POST /api/v1/upload/directory Upload multiple files

Upload multiple files in a single request as base64-encoded content. Creates one version (commit) containing all files.

Request Body
FieldTypeDescription
filesarray requiredNon-empty array of file entries
files[].pathstring requiredRelative file path
files[].content_base64string requiredFile content, base64-encoded
branchstring optionalBranch to commit to (default: main)
messagestring optionalCommit message (auto-generated if omitted)
parentsstring[] optionalParent version IDs (auto-resolves HEAD)
 
{
  "files": [
    {
      "path": "src/main.rs",
      "content_base64": "Zm4gbWFpbigpIHt9"
    },
    {
      "path": "Cargo.toml",
      "content_base64": "W3BhY2thZ2Vd..."
    }
  ],
  "branch": "main",
  "message": "upload project files"
}
{
  "version_id": "abc123...",
  "root": "def456...",
  "branch": "main",
  "stats": {
    "total_files": 2,
    "total_bytes": 8192,
    "total_chunks": 5,
    "new_chunks": 4,
    "dedup_chunks": 1,
    "dedup_bytes": 2048
  }
}

Download & Browse

GET /api/v1/version/{ref}/file/{*path} Download a file

Download a file's content as a binary stream. Reassembles chunks on-the-fly and streams the result.

Response: Binary stream with Content-Type: application/octet-stream, Content-Disposition: attachment, and Content-Length header.
NameTypeDescription
refstring requiredVersion ID, branch, or tag
*pathstring requiredFile path within the version tree
cURL Example
curl -o output.rs http://localhost:3000/api/v1/version/main/file/src/main.rs
